PUSH-оповещения

Для передачи оповещений от сервера на сторону мобильного приложения используются два механизма:

  • LiveTex SDK Core после успешной инициализации начинает постоянное соединение с сервисом оповещений LiveTex. Для коммуникации используется long-polling транспорт.
  • В случае неактивности мобильного приложения с SDK для доставки оповещений могут использоваться специфичные для платформы сервисы PUSH-оповещений. GCM в случае Android и APNS для iOS.
  • Также возможна интеграция, при которой оповещения отсылаются на сервера клиента.

Функциональность

  • Различные приложения и девелоперы могут выбирать разную стратегию оповещения.
  • Клиент может перенести всю логику оповещений на свою сторону.
  • Клиент может оставить всю логику оповещений на нашей стороне.
  • Стратегия оповещения может различаться для разных типов устройств. Может зависеть от типа платформы: Windows Phone, iOS, Android.

PUSH-нотификации на стороне клиента

При данном способе интеграции LiveTex передает на сервера клиентов оповещения.

PUSH-нотификации на стороне сервиса

При данном способе интеграции оповещения передаются через специфичные для платформы сервисы.
